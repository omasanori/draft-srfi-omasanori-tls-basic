== Abstract

This SRFI specifies a set of interfaces to establish secure connections to
peers and communicate with them using the Transport Layer Security (TLS)
protocol. It focuses on simplicity and ease to use over flexibility and
configurability, while it is intended to cover typical use cases.

== Rationale

As described in RFC 7258, large-scale surveillance is recognized as a privacy
threat nowadays, and the design of internet protocols has been shifting toward
encrypted-by-default, using TLS or similar protocols. Thus, a portable API for
TLS benefits programmers writing networking code.

Historically, TLS libraries have provided a number of functions. However, it
leads to bloat and misconfiguration, and ultimately, vulnerabilities. An API
for secure communications should not require programmers a lot of work for
achieving secure communications. This SRFI is significantly influenced by
libtls, a TLS library developed by the OpenBSD community "`as a response to the
unnecessary challenges other APIs present in order to use them safely.`" The
interfaces, however, provides the room for extensions by future SRFIs and
implementations.

This SRFI requires a socket API, including but not limited to, SRFI 106. It
assumes port I/O procedures are provided. It defines version flags for
implementations conforming to SRFI 176.
